## 🧱 Report Format

Each bug report includes:

- **Title** — concise summary of the issue
- **Description** — what the bug is and where it occurs
- **Severity & Priority** — with written justification, not just a label
- **Environment** — browser, URL, site under test
- **Preconditions** — state required before reproducing
- **Steps to Reproduce** — numbered, specific, repeatable
- **Expected vs Actual Result** — clearly separated
- **Suggested Fix** — practical recommendation for developers
- **Regression Test Cases** — follow-up tests to confirm the fix and prevent recurrence

---

## 💡 Why Severity ≠ Priority

This portfolio deliberately separates these two fields with justification for each, since they are commonly confused:

- **Severity** = how technically damaging the bug is (Critical / Major / Minor / Trivial)
- **Priority** = how urgently it should be fixed (High / Medium / Low)

A bug can be low severity but high priority (e.g., a typo on a homepage CTA) or high severity but low priority (e.g., a crash in a rarely used admin tool).
